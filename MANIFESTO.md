# KAIRO OS DESIGN & ARCHITECTURE MANIFESTO

## 1. Core Paradigm: The Premium PIOS
KAIRO is a Personal Intelligence Operating System (PIOS). It strikes a precise balance: architectural transparency meets elite product design. It surfaces system execution data with the elegance of modern macOS utilities or productivity engineering tools like Linear, Raycast, and Arc Browser.

## 2. Refined Product Language & Lexicon
* **Thread Registry:** The layout managing active or archived intelligence execution threads.
* **System Directive Console:** The command tray used to dispatch operational instructions to the Kernel.
* **Kernel Stream / Agent Output:** The real-time, compiled state stream generated by an active processing block.
* **Agent Firmware:** The baseline configurations detailing an agent's operational logic.
* **Active Page State / Local Cache:** The immediate memory address spaces currently visible to the active model.
* **Knowledge Vault:** The encrypted, local relational storage engine for long-term semantic knowledge.

## 3. Premium UI & UX Layout Philosophy
* **Spatial Cleanliness:** Interfaces are clean, utilizing distinct container panels separated by thin, high-contrast structural borders.
* **Contextual Metadata Surfaces:** Technical details exist as crisp, lowercase monospace labels that sit on the periphery of the main view.
* **Micro-Animations:** State transitions utilize precise easing curves (`cubic-bezier(0.16, 1, 0.3, 1)`) mimicking native desktop applications.

## 4. Premium Visual Specification & Typographic System
### The Brand Canvas Palette
* `--sys-bg`: #0A0F1C (Premium Deep Midnight Space)
* `--sys-surface`: #1B2433 (Dark Slate Panel Surface)
* `--sys-accent`: #3B82F6 (Premium Brand Blue)
* `--sys-text-primary`: #F8FAFC (Clean Off-White)
* `--sys-text-muted`: #D9E1EA (Soft Slate)
* `--sys-success`: #36D399 (Emerald Green)
* `--sys-border`: rgba(217, 225, 234, 0.1)

### Strict Typography Rules
1. **Primary Interface Typography (Satoshi):** Used for all core user text, agent narrative outputs, main menus, and input controls.
2. **Technical & Metadata Typography (JetBrains Mono):** Used exclusively for raw code blocks, system diagnostics, background log streams, active execution timestamps, thread IDs, and operational metrics.

## 5. Architectural Subsystems
* **The Kernel Core:** The hidden engine orchestrating local Express middleware, asynchronous state flow, and data management.
* **The Model Broker Control:** The internal interface driver communicating with Ollama to swap, monitor, or execute local model weights.
* **The Thread Registry:** The persistent UI drawer and database system tracking operational threads.
* **The Agent Registry:** The collection of specialized system behaviors available for orchestration.
* **The Runtime Monitor:** A subtle, collapsible diagnostic tray displaying behind-the-scenes logging data in monospace.
